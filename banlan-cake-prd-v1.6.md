## 2. 产品目标

泰斓 TAILAN 是一个面向家庭烘焙泰式斑斓甜品的微信小程序，采用“今天拼、明天取”的地铁站集中自提模式。

V1.6 要解决四个核心问题：

1. 用户能快速了解品牌、选择商品和自提站点，并完成拼团下单。
2. 商家能按每日生产能力配置各 SKU 库存、站点、自提时间和位置图片。
3. 系统能正确处理15分钟待支付库存预占、每站5件成团、22:00截单、次日12:00配送确认以及退款。
4. 商家和核销员能按站点完成备料、核销、迟到联系和固定地点放置交付。

### 2.1 本版本做什么

- 品牌展示首页。
- 商品分类、商品详情和 SKU 选择。
- 商家手动发布“今天卖、明天取”批次。
- 每个 SKU 独立设置批次库存，所有站点共享。
- 每站累计5件已付款商品达到成团门槛。
- 待支付订单预占库存15分钟。
- 当天22:00停止新下单。
- 取货日12:00按站点最终确认是否配送。
- 用户在订单完成交付前可以整单退款。
- 站点、自提时间、地点说明和图片管理。
- 6位自取码、二维码、截图代取和现场核销。
- 超级管理员、核销员权限隔离。
- MOCK 支付与退款流程。

### 2.2 本版本不做什么

- 不接入真实微信支付。
- 不支持部分退款，只支持整单退款。
- 不支持订单顺延或迁移到下一批次。
- 不做质量问题申请、图片举证和质量售后审核系统。
- 不做购物车和跨商品合并结算；一次下单基于一个已选 SKU 和数量。
- 不做团长奖励、周期购和复杂营销优惠。
- 不自动发布每日批次，必须由商家确认库存后手动发布。

## 3. 角色与权限概览

### 3.1 普通用户

- 可以不登录浏览首页、分类和商品详情。
- 付款前必须填写昵称和有效手机号。
- 可以选择商品、数量和站点。
- 可以创建待支付订单并完成 MOCK 支付。
- 可以查看订单、取货券、二维码和6位自取码。
- 可以在订单完成交付前申请整单退款。
- 可以把取货券截图、二维码或6位码交给可信任的人代取。

### 3.2 超级管理员 `superAdmin`

- 管理商品、SKU、价格和上下架状态。
- 管理站点池、线路、出口、地点说明和图片。
- 创建、编辑和手动发布批次。
- 设置每个 SKU 的批次库存。
- 为每个站点设置自提日期、开始时间、结束时间、位置说明和图片。
- 低于5件时手动确认配送。
- 手动关闭整个批次或单个站点。
- 处理退款重试。
- 查看所有站点备料和订单。
- 执行普通核销和异常跨站核销。
- 管理核销员权限。

### 3.3 核销员 `verifier`

- 只能查看被授权批次和站点。
- 可以查看对应站点备料清单和待核销订单。
- 可以扫码或输入6位码核销对应站点订单。
- 可以联系对应站点迟到用户。
- 可以上传固定地点放置证明。
- 不能管理商品、库存、批次、站点或退款。
- 不能普通跨站核销。

## 4. 核心业务参数

| 参数 | V1.6 规则 |
|---|---|
| 销售模式 | 今天卖，明天取 |
| 批次发布 | 商家手动发布 |
| 用户可见批次 | 同一时间最多一个接单中批次 |
| 截单时间 | 销售日北京时间22:00 |
| 22:00后销售 | 停止新下单，不销售后天取批次 |
| 成团统计单位 | 每站已付款商品总件数，不区分 SKU |
| 默认成团门槛 | 每站5件 |
| SKU库存 | 各 SKU 独立设置，批次内所有站点共享 |
| 待支付库存 | 预占15分钟 |
| 待支付是否计入成团 | 不计入 |
| 最终配送确认 | 取货日北京时间12:00 |
| 12:00不足5件 | 取消该站并退还全部剩余订单 |
| 12:00达到5件 | 确认配送 |
| 低于5件手动配送 | 允许，仅超级管理员，必须二次确认并填写原因 |
| 12:00后跌破5件 | 只要还有有效订单，仍继续配送 |

| 用户退款 | 完成交付前支持整单退款 |
| 完成交付 | 已核销，或商家完成固定地点放置并上传证明 |
| 未取货 | 不退款、不顺延 |
| 代取 | 支持截图、二维码或6位码代取 |
| 手机号 | 付款前强制填写 |

所有日期、22:00、12:00和15分钟超时计算必须使用北京时间 UTC+8。

## 8. 站点池与自提信息管理

## 8.1 站点池基础字段

超级管理员可以新增、编辑、启用或停用站点。

每个站点至少包含：

- `name`：站点名称。
- `line`：地铁线路。
- `exit`：出口。
- `pickupNote`：基础位置说明。
- `defaultLocationImages[]`：默认位置图片，最多3张。
- `status`：启用或停用。

主打站点为：

- 布吉站，3/14号线。
- 大学城站，5号线。

### 8.2 批次站点字段

商家创建批次时，从站点池选择开放站点，并为本批次填写：

- `thresholdN`：默认5件。
- `pickupDate`：取货日期。
- `arriveAt`：开始自提时间。
- `leaveAt`：结束自提时间。
- `locationNote`：本批次详细位置说明。
- `locationImages[]`：本批次位置图片，最多3张。

本批次位置说明和图片优先于站点池默认信息。

### 8.3 图片上传

- 管理员通过微信原生选择图片并上传云存储。
- 后台保存云文件 `fileID`。
- 图片上传失败不能发布不完整批次。
- 编辑窗口时保留已上传图片，支持删除和补传。
- 用户在选择站点页和取货券页查看图片。
- 图片不写入二维码；二维码只用于识别订单或自取码。

### 8.4 用户选择站点页

每张站点卡展示：

- 站点名、线路和出口。
- 已付款件数和5件门槛进度。
- 当前状态。
- 自提日期、开始时间和结束时间。
- 详细位置说明。
- 位置图片缩略图，点击可预览。

用户必须选择一个开放站点才能支付。

## 9. 批次创建、发布与销售日规则

## 9.1 草稿创建

商家可以随时创建下一销售日草稿，填写：

- 批次名称。
- 取货日期。
- SKU 与批次库存。
- 开放站点。
- 每站门槛、时间、地点和图片。

草稿不对用户展示，也不接受订单。

### 9.2 手动发布

- 批次只能由超级管理员手动发布。
- 发布前显示完整预览和二次确认。
- 发布日为 D 时，取货日期必须是 D+1。
- 截单时间自动设为 D 日北京时间22:00。
- 同一时间最多存在一个用户可见的接单中批次。
- 商家不发布时，用户只能浏览商品，不能下单。
- 不自动复制昨天库存，也不自动发布下一批。

### 9.3 22:00后规则

- 当天22:00立即停止新订单创建和支付。
- 已创建但尚未付款的订单全部超时，释放库存。
- 22:00后不销售后天取批次。
- 商家可以保存未来批次草稿，但不能在当天22:00后将其发布给用户。
- 第二天必须由商家重新确认库存并手动发布新的“今天卖、明天取”批次。

### 9.4 发布前服务端校验

必须满足：

- 批次名称不为空。
- 取货日期是发布日次日。
- 至少选择一个启用站点。
- 同一站点不能重复。
- 至少有一个上架 SKU。
- 每个 SKU 库存为非负整数或明确标记无限库存。
- 同一 SKU 不能重复。
- 每站门槛为正整数，默认5。
- 每站开始时间、结束时间、位置说明完整。
- 结束时间晚于开始时间。
- 每站位置图片不超过3张。

批次及其站点、窗口和库存必须全有或全无。推荐先完整写入草稿，全部成功后再把批次切换为接单中。

## 10. 库存、待支付预占与支付确认

## 10.1 共享库存模型

库存单位为 `batchId + skuId`。

例如：

- A商品本批库存20件。
- B商品本批库存8件。

布吉站和大学城站购买时共同扣减这20件和8件，不拆分站点库存。

### 10.2 库存字段

- `totalQty`：本批设定生产总量。
- `availableQty`：当前还可被新订单预占的数量。
- `reservedQty`：待支付订单临时占用数量。
- `soldQty`：已付款数量。
- `refundedQty`：已付款后退款数量，用于经营统计。
- `isUnlimited`：是否为无限库存。

必须满足：

`availableQty + reservedQty + soldQty - refundedQty = totalQty`

无限库存不参与有限库存售罄判断。

### 10.3 创建待支付订单

用户点击支付前，系统必须在事务中：

1. 校验批次仍在接单中。
2. 校验当前时间早于22:00。
3. 校验站点仍开放。
4. 校验手机号完整。
5. 校验 SKU 和数量。
6. 预占库存。
7. 创建待支付订单。
8. 写入 `expiresAt = createdAt + 15分钟`。

预占成功后：

- `availableQty` 减少。
- `reservedQty` 增加。
- 不增加站点已付款件数。
- 不触发成团。

### 10.4 支付成功

15分钟内支付成功后，在事务中：

- `reservedQty` 减少。
- `soldQty` 增加。
- 订单进入待配送确认。
- 站点 `paidItemCount` 增加商品件数。
- 判断是否达到5件门槛。

同一支付回调重复到达时，只能处理一次。

### 10.5 取消、超时和支付结果不明确

- 用户主动取消待支付订单，立即释放库存。
- 15分钟未付款，订单进入已超时并释放库存。
- 定时任务持续清理超时订单。
- 查询库存、创建订单和支付确认时也要懒清理已过期预占。
- 用户取消微信支付后，在15分钟有效期内可以重试。
- 支付结果不明确时先查订单状态，不重复创建订单。
- 22:00到达时，所有待支付订单立即超时并释放库存。

### 10.6 售罄

- 单个 SKU 售罄只关闭该 SKU。
- 其他 SKU 仍有库存时，批次继续接单至22:00。
- 只有所有有限且可售 SKU 都为0，才触发批次库存售罄。
- 库存售罄后停止新订单，并进入与手动截单相同的批次推进流程。

## 11. 每站5件成团与次日12:00配送确认

## 11.1 成团计数

- 成团按站点统计。
- 统计每个站点所有 SKU 的已付款商品件数之和。
- 不按 SKU 分别成团。
- 不按订单数量成团。
- 待支付预占订单不计入。
- 默认门槛为5件。

示例：布吉站用户分别购买 A×2、B×1、C×2，合计5件，达到门槛。

### 11.2 销售期间状态

- 已付款件数少于5：`拼团中`。
- 已付款件数达到5：`已达门槛待确认`。
- 达到门槛后仍可继续下单，直到22:00或库存售罄。
- 销售期间用户退款后，重新计算已付款件数。
- 截单前人数退款归零，站点仍保持拼团中，后续用户可以重新发起该站拼团。

### 11.3 22:00截单

22:00只停止新增订单，不立即把所有站点判定为最终成功或失败。

- 批次进入 `已截单待配送确认`。
- 所有待支付订单超时并释放库存。
- 各站保留当前已付款件数。
- 用户仍可以对未完成交付的订单退款。
- 不再允许新增订单补回件数。

### 11.4 取货日12:00自动确认

系统在取货日北京时间12:00逐站处理：

已经由超级管理员手动确认配送的站点不再按件数重新取消，12:00任务直接保留其已确认配送状态。

#### 已付款件数达到5

- 站点进入 `已确认配送`。
- 相关订单进入 `待自提`。
- 商家按站点备料并前往配送。
- 用户端展示明确自提时间、地点和图片。

#### 已付款件数不足5

- 站点进入 `关闭退款中`。
- 剩余有效订单全部整单退款。
- 退款完成后站点进入 `已关闭`。
- 商家不需要前往该站。

### 11.5 低于5件手动确认配送

超级管理员可以在12:00前手动确认低于5件的站点仍然配送。

操作要求：

- 二次确认。
- 展示站点和当前已付款件数。
- 必须填写原因。
- 记录操作人和时间。
- 操作后站点立即进入已确认配送。
- 已付款订单进入待自提。
- 取货日12:00自动任务不得再次把该站判定为不足5件关闭。

### 11.6 12:00后用户退款

- 用户仍可以在订单完成交付前整单退款。
- 退款后减少站点已付款件数并记录退款件数。
- 如果还有至少1件有效订单，站点保持已确认配送，商家继续交付。
- 不因跌破5件而取消剩余用户的配送。
- 如果所有用户都退款、有效件数变为0，系统自动取消该站配送。

## 12. 用户资料与手机号

### 12.1 浏览与下单边界

- 用户无需提供资料即可浏览。
- 付款前必须填写昵称和有效手机号。
- 手机号为空或格式不正确时，不能创建订单、预占库存或参加拼团。

### 12.2 当前手机号方案

当前为个人主体内部演示版，默认使用手动输入手机号。

- 不依赖微信一键获取手机号。
- 后续主体和接口条件满足后，可以增加用户主动授权的一键手机号能力。
- 一键能力失败时仍需保留手动输入。

### 12.3 手机号用途与可见范围

手机号仅用于：

- 自提联系。
- 用户迟到联系。
- 订单和地点异常通知。

权限：

- 超级管理员可以查看订单完整手机号。
- 被授权站点核销员可以查看或拨打本场用户手机号。
- 其他页面和其他站点默认脱敏。

## 13. 用户退款机制

## 13.1 基本原则

- V1.6 支持用户在订单完成交付前整单退款。
- 不限制为取货日上午某个固定时间前。
- 不支持部分商品退款。
- 不支持顺延或迁移到下一批。
- 已核销或已完成固定地点放置的订单，不再支持用户原因退款。

### 13.2 可以退款的状态

- 待配送确认。
- 待自提。
- 已确认配送但尚未核销、尚未完成固定地点放置。

待支付订单不走退款，直接取消并释放预占库存。

### 13.3 退款处理

退款必须使用统一流程：

1. 校验订单属于当前用户。
2. 校验订单尚未完成交付。
3. 使用订单 ID 生成稳定退款幂等键。
4. 把订单标记为退款处理中。
5. 回退已付款件数。
6. 记录退款件数并恢复批次库存账目。
7. MOCK 模式直接进入已退款。
8. 正式支付模式调用微信支付退款接口。
9. 正式支付以退款通知或退款查询结果为最终结果。
10. 失败时记录退款失败并支持使用相同退款单号重试。

### 13.4 退款与核销并发

退款和核销必须互斥：

- 退款事务先成功：自取码立即失效，不能核销。
- 核销事务先成功：订单已完成，不能退款。
- 固定地点放置事务先成功：订单完成交付，不能退款。

### 13.5 商家无法交付

商家无法交付时只支持退款，不支持顺延。

场景包括：

- 商家临时取消站点。
- 商家无法按承诺到达。
- 商品缺失或无法提供。

超级管理员关闭站点后，所有未完成订单进入退款流程。

### 13.6 当前版本不做质量售后

V1.6 不提供独立的质量问题申请、图片举证和商家质量审核流程。

用户仍可通过“联系商家”沟通，但不在本版本建立结构化质量售后状态机。

### 14.1 支付成功页

## 15. 取货券、代取与现场核销

## 15.1 取货券内容

- 站点名称、线路和出口。
- 自提日期和时间窗口。
- 详细位置说明。
- 位置图片。
- 商品和数量。
- 订单金额。
- 大号6位自取码。
- 二维码。
- 联系商家按钮。

### 15.2 截图代取

- 用户可以截图取货券，或把二维码、6位码交给可信任的人。
- 核销以有效自取码为准，不强制核验代取人身份。
- 第一次成功核销后立即失效。
- 用户应自行保管，因主动泄露或重复转发产生的提前核销风险由用户承担。

### 15.3 普通核销

核销员先选择当前批次和站点，再扫码或输入6位码。

后端必须检查：

- 订单存在。
- 订单属于当前批次。
- 订单属于当前站点团。
- 订单状态为待自提或已放置待自取。
- 订单没有退款。
- 自取码没有核销过。
- 核销员具有该站点权限。

成功后：

- 订单进入已完成。
- 写入核销时间。
- 写入核销人、方式、批次和站点日志。

### 15.4 异常跨站核销

- 普通核销员不能跨站核销。
- 超级管理员可以异常跨站核销。
- 必须展示正确站点和当前站点差异。
- 必须二次确认并填写原因。
- 核销日志必须标记为异常跨站核销。

## 16. 迟到、未取货与固定地点放置

### 16.1 迟到联系

自提窗口内用户未到时，商家或本场核销员可以通过订单手机号联系用户。

后台展示：

- 用户手机号。
- 拨打按钮。
- 联系状态和备注。

### 16.2 固定地点放置

商家准备离开时，可以把未领取商品放到预先约定的固定地点。

操作流程：

1. 选择未领取订单。
2. 填写或确认固定地点说明。
3. 上传现场照片。
4. 系统记录放置时间、站点和操作人。
5. 订单进入 `已放置待自取`。
6. 用户在取货券中看到地点、图片和放置时间。
7. 系统向已授权用户发送提醒。

### 16.3 交付责任

商家按约定完成联系、固定地点放置、拍照和系统记录后，视为完成交付。

- 放置后不支持用户原因退款。
- 不支持顺延。
- 商家不承诺持续看管，也不设置系统保留倒计时。
- 因用户迟取、现场管理或他人拿取产生的风险由用户承担。
- 固定地点必须经过允许并适合临时放置食品，不能随意占用地铁站公共区域。

### 16.4 结束本场

- 核销员只能结束被授权站点本场自提。
- 结束前必须处理所有待自提订单：已核销或已放置待自取。
- 不能直接把未处理订单批量变成未取货终态。
- 超级管理员可以异常提前结束，但必须二次确认并填写原因。

## 17. 商家工作台

## 17.1 工作台概览

展示：

- 当前接单批次。
- 已截单待配送确认批次。
- 各站已付款件数和5件门槛。
- 各站12:00确认结果。

- 当天自提和核销进度。
- 待放置订单。

### 17.2 批次操作

- 创建草稿。
- 编辑库存、站点、时间和图片。
- 手动发布。
- 手动关闭整个批次。
- 查看操作记录。

### 17.3 站点操作

- 低于5件手动确认配送。
- 修改自提窗口和位置图片。
- 关闭单个站点并退款。
- 标记到达。
- 进入备料和核销。

### 17.4 手动关闭

“关闭整个批次”和“关闭单个站点”是两个独立操作。

共同要求：

- 仅超级管理员。
- 二次确认。
- 必须填写原因。
- 记录操作人、时间、批次和站点。
- 影响订单进入统一退款流程。
- 重复操作必须幂等。

### 17.5 修改时间和地点

- 批次发布前可以自由修改。
- 已有付款订单后，必须填写修改原因。
- 修改日期、站点、出口或时间变化超过30分钟，视为重大变化。
- 重大变化必须通知已付款用户。
- 如果商家无法按修改后的承诺履约，只能退款，不顺延。

## 18. 状态机

## 18.1 批次状态 `batches.status`

| 状态 | 说明 | 退出事件 |
|---|---|---|
| 草稿 | 商家配置库存、站点和窗口，用户不可见 | 手动发布或删除 |
| 接单中 | 用户可创建订单，截止到22:00 | 22:00、库存售罄、手动关闭 |
| 已截单待配送确认 | 停止新订单，等待取货日12:00 | 12:00处理全部站点 |
| 配送进行中 | 至少一个站点已确认配送 | 全部站点完成或关闭 |
| 关闭退款中 | 商家关闭批次，正在退款 | 所有退款处理完成 |
| 已结束 | 全部站点完成或退款关闭 | 终态 |

## 18.2 站点团状态 `batchStations.status`

| 状态 | 说明 |
|---|---|
| 拼团中 | 已付款件数不足5 |
| 已达门槛待确认 | 已达到5件，等待12:00或商家手动确认 |
| 已确认配送 | 12:00达到5件或商家手动确认，商家承担交付承诺 |
| 关闭退款中 | 不足5件或商家关闭，正在退款 |
| 已关闭 | 有效订单为0或退款完成，不配送 |
| 自提进行中 | 商家已到达并开始核销 |
| 已完成 | 本场所有有效订单已核销或已放置 |

## 18.3 订单状态 `orders.status`

| 状态 | 说明 |
|---|---|
| 待支付 | 预占库存15分钟，不计入成团 |
| 已取消 | 用户取消，库存已释放 |
| 已超时 | 15分钟或22:00超时，库存已释放 |
| 待配送确认 | 已付款，等待站点12:00确认 |
| 待自提 | 站点已确认配送 |
| 退款处理中 | 已发起整单退款 |
| 已退款 | 退款流程完成 |
| 已放置待自取 | 商家完成固定地点放置和拍照，视为已交付 |
| 已完成 | 已现场核销 |

旧版的截单过渡、未取货待处理、顺延和支付后退款中等状态不再作为 V1.6 订单状态；实现时必须统一迁移到本节定义的状态集合。

## 18.4 退款状态 `refunds.status`

| 状态 | 说明 |
|---|---|
| 退款处理中 | MOCK处理或等待微信退款结果 |
| 退款失败 | 调用失败，允许同退款单号重试 |
| 已退款 | MOCK完成或微信确认成功 |

本版本没有用户质量售后审核，因此不使用待审核、已驳回等质量售后状态。

## 20. MOCK模式与正式支付上线门槛

### 20.1 当前 MOCK 规则

- `MOCK_PAY = true`。
- 支付按钮模拟支付成功。
- 退款模拟立即成功。
- 用于验证库存、成团、页面、核销和退款状态。
- 不代表已具备正式收款能力。

### 20.2 正式支付上线前必须完成

1. 取得可用微信支付商户号。
2. 前端真实拉起微信支付。
3. 后端校验合法支付回调。
4. 校验商户号、订单号、金额和支付状态。
5. 支付回调幂等。
6. 接入微信支付退款API。
7. 接收并校验退款结果通知。
8. 退款查询和失败重试。
9. 使用稳定退款单号避免重复退款。
10. 完成真实小额支付、取消、超时、退款和重复回调联调。

以上事项必须长期保留在项目说明和上线前待办中，不能因为 MOCK 流程跑通而删除。

## 21. 异常和恢复规则

### 21.1 重复创建或支付

- 支付按钮请求期间禁用。
- 同一订单重复支付只处理一次。
- 支付结果未知时先查订单，不重新占库存。

### 21.2 预占释放失败

- 超时订单保持待清理标记。
- 定时任务反复重试。
- 释放必须幂等，不能增加超过总库存。

### 21.3 12:00确认任务失败

- 批次保持已截单待配送确认。
- 记录未处理站点和错误。
- 定时器重试，已处理站点不重复处理。

### 21.4 批量退款中断

- 每个订单独立退款幂等。
- 站点保持关闭退款中。
- 所有订单完成后才进入已关闭。

### 21.5 通知失败

- 不影响订单主流程。
- 记录失败并允许管理员查看。
- 用户仍可在订单和取货券中查看最新信息。

### 21.6 图片上传失败

- 不保存无效 `fileID`。
- 发布批次前必须检查图片列表。
- 上传失败允许重试，不丢失其他表单内容。

## 22. 验收标准

## 22.1 文档与文件完整性

- JS 逐文件语法检查无错误。
- JSON 全部可解析。
- UTF-8、BOM、NULL、文件截断和 WXML 标签扫描无损坏。
- 前端调用 action 与云函数路由一致。
- 页面不得出现旧版门槛、旧版退款截止时间或顺延操作入口。

## 22.2 首页与点单

- 首页不展示站点卡和站点进度。
- 首页只有一个主操作“立即去拼”。
- 首页保持品牌展示和底部 tabBar。
- 点单页不展示完整站点卡。
- 用户先选商品，再进入独立选站页。

## 22.3 批次发布

- 商家可以保存草稿。
- 只有超级管理员可以发布。
- 发布日与取货日相差1天。
- 截止时间固定为发布日22:00。
- 22:00后不能继续接单或发布当天销售批次。
- 同时只有一个用户可见接单批次。

## 22.4 站点与图片

- 后台可设置站点、线路、出口、日期、时间、说明和最多3张图片。
- 选站页和取货券展示正确图片。
- 修改图片不会破坏已有窗口信息。
- 未完整配置的站点不能发布。

## 22.5 待支付库存

- 创建待支付订单立即预占库存。
- 待支付不计入成团。
- 15分钟内支付成功，预占转已售。
- 15分钟超时释放库存。
- 22:00时所有待支付订单超时。
- 重复点击不重复占库存。
- 两个站竞争最后一份库存只能有一个成功。

## 22.6 成团与12:00确认

- 每站不同 SKU 已付款件数合计达到5件时显示已达门槛。
- 22:00停止新增订单。
- 取货日12:00达到5件，确认配送。
- 取货日12:00不足5件，剩余订单全部退款。
- 超级管理员可以低于5件手动确认配送并填写原因。
- 已手动确认配送的站点不会在12:00被重新取消。
- 12:00后退款跌破5件但仍大于0，继续配送。
- 12:00后全部退款至0件，自动取消该站。

## 22.7 退款

- 用户在核销或固定地点放置前可以整单退款。
- 退款后减少站点件数并记录退款库存。
- 退款与核销并发时只有一个操作成功。
- 同一订单重复退款只产生一笔有效退款。
- MOCK模式直接进入已退款。
- 正式支付接入后以退款通知或查询结果为准。

## 22.8 手机号

- 用户不填手机号不能创建订单和预占库存。
- 手机号格式错误有明确提示。
- 用户仍可在未填手机号时浏览。
- 核销员只能看到授权站点用户手机号。

## 22.9 核销与放置

- 普通核销员只能核销授权站点订单。
- 错站码明确提示正确站点。
- 超级管理员跨站核销需要确认和原因。
- 二次核销被拒绝。
- 固定地点放置必须有地点、图片、时间和操作人。
- 放置后订单不再支持退款。
- 结束本场前不存在未处理待自提订单。

## 22.10 权限

- 普通用户不能绑定超级管理员。
- 核销员不能管理商品、库存、批次、站点或退款。
- 整批关闭和单站关闭分开。
- 两种关闭都要求超级管理员、二次确认和原因。
