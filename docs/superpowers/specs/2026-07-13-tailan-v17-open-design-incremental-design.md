# 泰斓 V1.7 Open Design 增量设计规格

## 目标

在完整保留 `设计稿v3-解包版.html` 的前提下，生成独立的 `设计稿v4-V1.7.html`。新版继续使用 v3 的单文件横向画布、390×844 手机框、C 端明亮主题和商家端深色主题，仅增量落实 PRD V1.7 的页面、状态、文案和交互。

## 真源与优先级

1. 业务规则：`banlan-cake-prd-v1.7.md`。
2. 接口语义：`cloudfunctions/api/ACTIONS.md`。
3. 视觉与组件：`设计稿v3-解包版.html`。
4. 本规格负责页面拆分、状态表达和验收边界。

冲突时按上述优先级处理。v3 仅是视觉真源，不得让旧业务文案覆盖 V1.7。

## 实施方式

采用 Open Design + Codex 的混合增量流程：Open Design 读取 v3、PRD 和接口契约，沿用已有设计语言生成页面结构与视觉方案；Codex 将结果整合为一个离线可打开的 HTML，并完成规则、跳转和结构校验。

不覆盖 v3，不重画整套视觉，不把每个状态拆成独立页面。首页三态、支付三态和订单状态通过页面内部控件切换。

## 页面结构

新版按 0～15 编号，共 16 个主页面：

0. 首页：开团中、今日休息、未开团三态。
1. 商品分类：5 人门槛、购物车角标。
2. 商品详情：加入购物车、立即购买。
3. 购物车：新增。
4. 选择自提站点：按人进度、交付模式徽标。
5. 结算及手机号：新增。
6. 发起支付及 3 分钟预占：新增，包含成功、取消、超时反馈。
7. 支付成功：按人进度、两个订阅模板。
8. 站点团页：按人进度和分享落地。
9. 我的订单：四类履约状态、两段式退款入口。
10. 订单详情及取货券：手机尾号后 4 位、二维码、交付模式、现场照片。
11. 我的：资料、手机号、规则及身份入口。
12. 商家工作台：休息开关、催开团、配送确认/取消、退款申请。
13. 商品管理：即时改价、引用后只能下架。
14. 批次发布：勾选 SKU、填写库存、勾选站点、10:00 软提醒。
15. 现场核销/无人放置：扫码、尾号、重复预警、拍照和整场放置。

## 视觉约束

- 主色 `#3E8E5E`，亮绿 `#7BC96F`，浅绿底 `#EDF7EC`。
- 椰奶米白 `#FAF7EE`，奶油 `#FFF3DD`，价格暖橙 `#F08A3C`，紧迫色 `#E8722A`。
- 墨绿字 `#243329`，次绿 `#2C6B47`，正文 `#57614F`，辅助 `#8A8574`。
- 分隔线仅用 `#EAE4D6`、`#F0ECE1`；品类栏底 `#F5F1E6`。
- 商家端深色底 `#161D18`、`#1C2620`，浅绿字 `#D8E8CF`。
- 字体 `-apple-system, BlinkMacSystemFont, "PingFang SC", sans-serif`。
- 28px 大标题 800；20px 卡片标题 800；16px 正文 600；14px 正文 `#57614F`；12px 辅助 `#8A8574`；价格 22px/800/`#F08A3C`。
- 卡片圆角 20px，内容块 14px，按钮和标签 999px。
- 手机框 390×844，圆角 48px，10px `#171B18` 边框。
- 卡片阴影 `0 4px 14px rgba(40,60,45,.08)`。
- C 端明亮、商家端深色；禁止引入新审美和新组件体系。

## 关键业务表达

- 成团统一显示“已拼 X/5 人”；库存继续使用“份/个”。
- 支付预占固定 3:00，明确后台预占、没有待支付列表、超时或取消需重新发起。
- 取货凭证使用手机号尾号后 4 位，保留二维码。
- 站点明确标注“有人核销”或“无人放置”。
- 商家完成核销或放置前必须上传现场照片；用户取货券展示照片。
- 交付前按钮为“整单退款”，交付后按钮为“申请退款”。
- 支付成功页分别展示“成团结果通知”和“取货提醒”授权。
- 首页三态均可在同一屏切换；未开团态包含催开团按钮和已催人数。

## 交互主线

首页 → 商品分类 → 商品详情 → 购物车或立即购买 → 选择自提站点 → 结算及手机号 → 发起支付 → 支付成功 → 我的订单 → 订单详情及取货券。

所有顶部导航、按钮和底部入口使用锚点链接或同页状态切换，离线打开时可用。

## 交付物

- `设计稿v4-V1.7.html`：完整可浏览的单文件高保真设计稿。
- 页面顶部提供 0～15 快速导航。
- 文件尾部或页面说明区提供“相对 v3 修改页面”和“V1.7 新增页面”清单。

## 验收标准

1. 原 `设计稿v3-解包版.html` 字节不变。
2. v4 为 UTF-8 单文件，无外网运行依赖。
3. 16 个手机框尺寸、编号和标题完整。
4. 主线跳转没有失效目标。
5. 首页、支付、订单等状态切换可操作。
6. 页面不存在“15分钟”“6位核销码”“满5件成团”等旧规则。
7. “人”和“份/个”使用场景正确。
8. C 端和商家端视觉与 v3 一致。
9. HTML 标签结构完整，脚本无语法错误。
