# 泰斓 V1.7 同版上线验收

## 同版原则

小程序代码、`api` 云函数、数据库演示/迁移脚本必须来自同一 Git 提交。禁止先部署后端再临时修改前端，或线上补丁式混版。

## 发布前门禁

1. 记录待发布提交 SHA，确认工作树干净。
2. 备份云数据库；保留 `admins/users/config`，业务集合按迁移方案处理。
3. 执行完整自动验收：完整性、全部测试、全部 JS、JSON、WXML、UTF-8/NUL、action 交叉扫描。
4. 微信开发者工具真机冒烟：立即购买、多 SKU 购物车、取消/超时、支付成功、交付前退款、交付后申请、尾号重复候选、扫码核销、无人整场放置、未取收尾、权限隔离。
5. 配置两条订阅模板；正式支付环境必须完成商户配置、回调验签和退款对账。

## 发布顺序

1. 维护窗口开始，阻止商家继续发布新批次。
2. 备份并迁移数据库。
3. 部署同一提交的 `cloudfunctions/api`（云端安装依赖）。
4. 验证 `lifecycleTick` 定时器和环境变量 `SUPER_ADMIN_OPENID`。
5. 仅在演示环境通过可信任务运行 `initDemo`；正式环境不得覆盖业务数据。
6. 上传同一提交的小程序，版本号 `v1.7.0-internal`；完成真机冒烟后再提交审核。

## 立即回滚条件

- 多 SKU 库存非原子、3 分钟释放失效、按人计数错误。
- 22:00 提前退款或 12:00 配送判断错误。
- 普通用户越权、核销员跨站、退款重复执行。
- 无照片也能完成核销/放置，或用户看不到交付照片。

触发任一条件立即停止发布并恢复备份与上一稳定版本，不在线上现场改代码。
