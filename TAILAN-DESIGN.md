## 2. 字体与文字层级

字体族固定为：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", "Noto Sans SC", sans-serif`。输入控件继承同一字体，不引入新展示字体。

| 层级 | 字号 / 字重 | 行高 | 颜色与用途 |
|---|---|---|---|
| 页面大标题 | 28px / 800 | 多行约 1.25–1.35 | `#243329`，页面主标题 |
| 重点标题 | 22–24px / 800 | 1.25–1.35 | Hero、金额、结果 |
| 卡片标题 | 20px / 800 | 默认 | `#243329` |
| 模块标题 | 18px / 800 | 默认 | `#243329` |
| 操作/强调 | 16px / 600–800 | 默认 | 顶栏、按钮、强调正文 |
| 正文 | 14px / 400–700 | 段落 1.5–1.7 | `#57614F` |
| 列表正文 | 13px / 600–800 | 默认 | `#243329` / `#57614F` |
| 辅助说明 | 12px / 400–700 | 长说明 1.35–1.8 | `#8A8574` |
| 微型标签 | 10–11.5px / 600–800 | 默认 | 徽标、英文副标 |
| 价格 | 22px / 800 | 默认 | C 端 `#F08A3C` |

主要字重为 600、700、800；不得用大量细字重制造新风格。数字、金额、人数、件数保持高字重。

## 3. 画布、手机框与编号

- 横向连续画布背景为 `#E8E3D6`，手机框横向排列，框间距约 44px。
- 每屏上方显示“编号徽标 + 页面标题”；顶部总标题和快速导航只属于设计稿浏览层。
- 手机内容尺寸 `390px × 844px`，外框 `10px solid #171B18`，圆角 48px，`overflow:hidden`。
- C 端框背景 `#FAF7EE`，阴影 `0 24px 48px rgba(23,40,28,.18)`。
- 商家端框背景 `#161D18`，阴影 `0 24px 48px rgba(23,40,28,.30)`。
- 编号格式为连续阿拉伯数字 + 中文页面名，可在括号内补结构或用途，如 `3 选自提站点（选站 = 参团）`。
- 编号徽标最小 26×26px、圆角 8px、13px/800。C 端为 `#243329` 底、`#F3EFE2` 字；商家端为 `#1C2620` 底、`#7BC96F` 字。
- V1.7 使用 0–15 连续编号，继续复用同一标题格式。

## 4. 组件规则

### 卡片与内容块

- C 端主卡片：白色或米白底、圆角 20px、内边距 16px。
- 标准阴影：`0 4px 14px rgba(40,60,45,.08)`；轻列表可用透明度 `.06`。
- 强调卡可复用 `0 6px 18px rgba(40,60,45,.08)` 或 `0 12px 30px rgba(40,60,45,.14)`，不得继续加重。
- 选中卡片：`2px solid #3E8E5E`，可配 `0 8px 20px rgba(62,142,94,.18)`。
- 商家端卡片：`#202A22`、圆角 20px、内边距 16px，以明度差和透明白边界分层。
- 标准内容块圆角 14px；输入和紧凑面板 10–12px；较大输入 16px。
- C 端输入使用白/米白表面、浅分隔、`#243329` 正文和 `#B9B29F` 占位。
- 商家端输入使用 `#161D18`、`1px solid rgba(255,255,255,.12)`、`#E9F2EA` 文字。

### 胶囊按钮

- 圆角固定 999px；既有高度 44、46、48、50px，主要触控按钮不得低于 44px。
- C 端主按钮：`#3E8E5E`、白字、15–16px/700–800，可配 `0 6px 16px rgba(62,142,94,.28)`。
- C 端次按钮：白底、`1.5px solid #3E8E5E`、`#2C6B47` 字。
- 禁用按钮：`#F5F1E6` 底、`#B9B29F` 字、无强调阴影。
- 商家端主按钮：`#7BC96F` 底、`#161D18` 字、800 字重。
- 不新增硬角或渐变主按钮。

### 状态标签

- 圆角 999px，典型内边距 4px 10px，10.5–12px/700–800。
- 进行中：`#EDF7EC` 底 + `#2C6B47` 字。
- 已完成/已成团：`#3E8E5E` 底 + 白字。
- 紧迫：`#FFF0E2` 底 + `#E8722A` 字。
- 未开放/禁用：`#F5F1E6` 底 + `#9A9483` 或 `#B9B29F` 字。
- 商家端使用透明亮绿或透明白底，不直接搬用 C 端米白块。

### 进度条

- 轨道高 9–10px、圆角 999px、裁切溢出。
- C 端轨道 `#EAE4D6`；进行中填充 `#7BC96F`，完成后可切 `#3E8E5E`。
- 商家端轨道 `rgba(255,255,255,.08)`，填充 `#7BC96F`。
- 成团文案固定为“已拼 X/5 人”；库存和备料另用“X 份 / X 个”。

### 底部导航

- 白底、顶边 `1px solid #EFEAE0`、总高约 80px（含 16px 底部安全留白）。
- 每项等分，图标约 24px，图标与文字间距 3px。
- 激活态 `#3E8E5E`，11px/700；未激活态 `#A8A393`，11px/600。
- 使用 1.8–2px 线性 SVG；激活项只允许轻浅绿填充。
- V1.7 四项“首页 / 点单 / 我的订单 / 我的”复用同一结构，只增加“点单”入口，不另建导航组件。

## 5. 明亮主题与深色主题边界

- C 端范围：首页、分类、详情、购物车、选站、结算、支付、站点团页、订单、取货券、我的。以 `#FAF7EE`、`#FFFFFF`、`#EDF7EC` 为主，主操作 `#3E8E5E`，价格 `#F08A3C`，紧迫 `#E8722A`。
- 商家端范围：工作台、商品管理、批次发布、配送确认、退款处理、现场核销/无人放置。背景 `#161D18`，卡片 `#202A22`，主文字 `#E9F2EA`，主操作 `#7BC96F`。
- 深色主题仅用于管理与履约工具，不作为 C 端装饰；两端共享结构、圆角与间距，但必须按主题映射颜色。

## 6. 间距、阴影、圆角与图标

- 页面左右留白 20px；v3 局部使用 16–18px，新页优先 20px。
- 卡片内边距 16px，强调卡可为 18px；组间距 12/16/24px；内部常用 6/8/10px。
- 底部固定操作区常用 `12px 18px 26px`，保留安全区。
- 圆角：手机 48px；主卡 20px；Hero 24px；内容块 14px；输入 10–12px；编号 8px；按钮/标签/进度 999px。
- 阴影：标准卡 `0 4px 14px rgba(40,60,45,.08)`；主按钮 `0 6px 16px rgba(62,142,94,.28)`；商家端主要靠色阶与边界，不给每卡加阴影。
- 图标为简洁线性 SVG，`stroke-width:1.8–2`，圆端圆角；常规 16–24px，主要动作可到 28px。
- C 端激活图标 `#3E8E5E`，未激活 `#A8A393` / `#57614F`；商家端用 `#E9F2EA`、`#8FA491`、`#7BC96F`。
- 不引入新图标库、拟物图标、彩色 Emoji 或不同线宽体系。

## 7. 禁止事项

1. 不得引入新审美、新字体、新渐变主视觉或第二套组件体系。
2. 不得改变主色 `#3E8E5E`、亮绿 `#7BC96F` 及主题边界。
3. 不得混淆“人”和“份/个”：成团、门槛、催开团只用“人”；SKU 库存、购买量、备料只用“份”或“个”。
4. 不得继承 v3 的旧规则：以商品份数判定成团、旧预占时长、旧核销码、取货日固定时点后不可退款、顺延或跨批次迁移。
5. 不得把商家端深色主题扩散到 C 端，也不得把 C 端米白卡直接贴进商家端。
6. 不得为 V1.7 新状态另建按钮、标签、卡片或导航视觉体系；先映射既有组件变体。
7. 新增实现应先映射本文 Token；局部色必须来自 v3 已有值并注明用途。

## 8. V1.7 增量复用

| 增量页面/状态 | 复用组件 | 复用方式 |
|---|---|---|
| 首页三态 | 首页主卡、标签、主按钮、规则行 | 只替换营业状态、催开团计数和按钮状态，不重做 Hero |
| 购物车 | 商品列表卡、数量步进、底部操作区 | 增加勾选和失效态 |
| 结算及手机号 | 选站卡、内容块、输入、底部主按钮 | 组合联系人、手机号与订单摘要 |
| 支付与 3:00 预占 | 订单卡、强调文字、主次按钮 | 取消/超时沿用次按钮和禁用态 |
| 支付成功双订阅 | 成功结果卡、通知开关 | 两个模板作为同组件两行实例 |
| 站点按人进度 | 选站卡、团页进度、标签 | 统一“已拼 X/5 人”，库存独立展示 |
| 两段制退款 | 订单卡、标签、胶囊按钮 | 交付前“整单退款”，交付后同位置“申请退款” |
| 手机尾号取货券 | 票根、二维码区、强调数字 | 6 位码位改为手机号尾号后 4 位，保留二维码布局 |
| 交付模式 | 状态标签 | 展示“有人核销”或“无人放置” |
| 配送确认/取消 | 深色卡、警告提示、主次操作 | 复用工作台层级，不新建控制台 |
| 交付拍照 | 深色表单、虚线添加区、主按钮 | 复用既有添加态和内容块 |
| 尾号重复预警 | 暖橙提示、候选列表 | 用 `#F0A85C` 提示，列表内人工区分 |
| 批次发布 | 勾选、库存输入、发布按钮 | 删除按站设置门槛，加入 10:00 软提醒 |
| 无人放置 | 现场核销深色页、照片区、底部按钮 | 同一履约组件切换动作与状态 |

复用顺序：先复用布局骨架与主题映射，再复用卡片、内容块、标签、按钮、进度和导航；业务差异通过文案、数据和状态表达。v3 无对应结构时，只能组合最接近的既有组件，并继续遵守本文尺寸、圆角、色彩和字重。V1.7 始终保持 390×844 手机框、横向画布和连续编号体系。
