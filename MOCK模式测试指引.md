# MOCK 模式测试指引（V1.7）

适用前提：云函数环境未设置 `MOCK_PAY=false`（默认 true）。此模式只验证内部演示链路，不调用真实微信支付，不能证明正式支付、真实退款或订阅消息真机发送已完成。

## 1. 本地自动验收

在不部署、不清库、不运行 `initDemo` 的本地场景，先跑：

```powershell
node scripts/check-integrity.js
node --test cloudfunctions/api/tests/*.test.js scripts/tests/*.test.js
```

预期：完整性检查 `INTEGRITY PASS`，全部测试通过。检查范围包括 JS/JSON/WXML/UTF-8、前后端 action 对齐、旧规则残留、V1.7 订单/批次/履约/退款/购物车/通知 outbox 规则。

## 2. 安全与环境

1. `MOCK_PAY=true` 时，支付按钮模拟成功，退款模拟处理；不要把它当作真实收款能力。
2. `MOCK_PAY=true` 默认开放演示商家入口：任意测试微信可获得请求内临时 `superAdmin`，不写入 `admins` 集合。
3. 如需在 MOCK 下恢复正式权限校验，设置云函数环境变量 `DEMO_OPEN_ADMIN=false` 后重新部署 `api`。
4. `MOCK_PAY=false` 时，演示商家入口强制关闭，即使误设 `DEMO_OPEN_ADMIN=true` 也不能绕过管理员权限。
5. 演示数据只能由可信云端任务执行 `initDemo` 初始化；正式环境不得清库或覆盖业务数据。

## 3. 下单与 3 分钟预占

1. 路径：点单/商品详情 → 立即购买，或商品详情 → 加入购物车 → 购物车勾选 → 去结算。
2. 结算必须填写联系人和有效手机号；手机号尾号后 4 位作为现场取货凭证。
3. 创建订单后检查：订单为后台隐藏的 `预占中`，库存从 `availableQty` 转入 `reservedQty`，站点成团人数不增加。
4. 3 分钟内 MOCK 支付成功后检查：订单进入 `待配送确认` 或 `待自提`，预占转 `soldQty`，站点按不同付款 openid 计 1 人。
5. 同一用户多单或多件仍只计 1 人；库存和备料仍按件统计。
6. 取消支付或超过 3 分钟后，订单变为 `已取消` 或 `已超时`，预占库存只释放一次。
7. 22:00 后不允许继续创建或支付订单，未付款预占应释放。

## 4. 22:00 与取货日 12:00

1. 触发受信 `lifecycleTick` 到销售日 22:00：只截单并锁定各站按人数的成团结果，不在 22:00 直接退款或关站。
2. 触发到取货日 12:00：达到 5 个不同付款用户的站点确认配送；不足 5 人的站点进入关闭退款流程。
3. 22:00 后、12:00 前，超级管理员可填写原因执行 `manualConfirmDelivery`，确认不足 5 人仍配送。
4. 已手动确认配送的站点不会在 12:00 被重新取消。
5. 成团后个别退款不散团；有效订单退到 0 人时才自动取消该站。

## 5. 用户退款与交付后申请

1. `待配送确认` 或 `待自提` 订单可由用户自助整单退款。
2. MOCK 模式下自助退款应进入 `已退款`，库存回补，站点人数按剩余有效付款用户重新计算。
3. `已完成`、`已放置待自取`、`已完成未取` 不允许自助退款，只能提交 `applyRefundRequest` 进入人工个例处理。
4. 超级管理员通过 `resolveRefundRequest` 批准或驳回交付后申请；批准时仍按整单退款处理。
5. 不支持部分退款、跨批次迁移或订单顺延。

## 6. 现场履约

1. 核销员工作台只能看到授权批次/站点及对应手机号；普通用户不能进入商家功能。
2. `已确认配送` 站点点击到达后进入 `自提进行中`。
3. 核销支持扫二维码、输手机尾号和候选订单人工选择；尾号必须为 4 位数字。
4. 同场次尾号重复时，后台返回候选订单，核销员按姓名、手机号和商品人工确认。
5. 核销、无人放置、有人核销未取收尾都必须上传 1-3 张交付照片；无照片必须失败。
6. 普通核销员不能跨站核销；超级管理员异常跨站必须二次确认并填写原因。
7. 无人放置站执行 `placeOrderAtLocation` 后，订单进入 `已放置待自取`，视为交付完成。
8. 有人核销窗口结束仍未取的订单，可拍照挪货后执行 `finishNoShow`，进入 `已完成未取`，不自动退款。
9. `endPickupSession` 前必须没有未处理的 `待自提` 订单。

## 7. 必查旧规则

本地扫描和人工检查不得出现以下旧行为入口：

- 预占按 15 分钟处理。
- 用 6 位自取码作为核销凭证。
- 按件数或订单数成团。
- 22:00 直接退款或关站。
- 取货日 10:00 后不可退款。
- 支持顺延、延期到下一批或跨批次迁移。
- 使用废弃 action：`bindAdmin`、`createBatch`、`manualFormGroup`、`manualCutoff`、`extendDeadline`、`markNoShowOrders`、`markOrderPostponed`、`reviewRefund`。

## 8. MOCK 不能覆盖的项目

以下必须等云端、真机或正式资料齐全后单独验证：

- 微信开发者工具编译和真机操作。
- 云函数同版部署、环境变量和定时器真实触发。
- 云数据库备份、迁移、清库和 `initDemo`。
- 真实微信支付、支付回调验签、迟到成功兜底。
- 真实退款回调、查询、重试和对账。
- 两条订阅消息模板 ID 配置后的真机发送。
