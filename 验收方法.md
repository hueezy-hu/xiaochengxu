# V1.7 本地验收方法

本文件用于「不含真机、不部署、不改云数据」的本地验收。正式上线仍以 `V1.7-上线验收.md`、云端同版部署和真机冒烟为准。

## 1. 自动检查

依次运行：

```powershell
node scripts/check-integrity.js
node --test cloudfunctions/api/tests/*.test.js scripts/tests/*.test.js
```

预期全部通过。完整性检查覆盖 JS 语法、JSON 解析、WXML 标签、UTF-8/NULL、旧规则残留和前后端 action 对应关系。

## 2. 语义扫描

自动检查通过后，再人工关注这些关键词命中是否只是历史说明或禁止项：

- 旧预占：15 分钟、待支付列表。
- 旧核销：6 位自取码、verifyCode。
- 旧成团：按件成团、累计 5 件。
- 旧退款：取货日 10:00 后不可退款、单段审核退款。
- 旧履约：顺延、跨批次迁移。
- 废弃 action：`bindAdmin`、`createBatch`、`manualFormGroup`、`manualCutoff`、`extendDeadline`、`markNoShowOrders`、`markOrderPostponed`、`reviewRefund`。

源码和页面不应保留这些旧入口；PRD 的历史对比、AGENTS 的禁止规则和测试断言可以保留。

## 3. 边界与竞争

重点看测试是否覆盖：

- 北京时间 21:59:59 / 22:00 下单与支付边界。
- 取货日 11:59:59 / 12:00 逐站确认边界。
- 最后一件库存的并发预占。
- 支付与 3 分钟超时竞争。
- 取消预占只释放一次。
- 同一用户多单多件仍只计 1 人。
- 用户退款与核销/固定放置竞争。
- 退款失败后保持 `关闭退款中` 并可由生命周期续跑。

## 4. 页面路径检查

不真机时至少做静态路径检查：

`首页 → 点单 → 商品详情 → 购物车/立即购买 → 选站 → 结算 → 支付成功 → 我的订单 → 订单详情/取货券`

再检查我的、站点团分享页、商家工作台、商品管理、批次发布、站点资料和现场核销页。用户必须先选商品，再选站点；只有可取货状态订单才能展示尾号凭证或二维码。

## 5. 权限与隐私

- 普通用户不能初始化演示数据或自绑定管理员。
- MOCK 演示入口只产生请求内临时超管，不写入 `admins` 集合。
- 核销员只看到授权站点与对应手机号。
- 普通核销员不能跨站；超级管理员跨站必须确认、原因和日志。
- 核销员不能进入商品、库存、批次、站点管理或退款处理。

## 6. 本地验收边界

本地通过只能说明代码和文档门禁通过，不能声明正式上线。以下仍需单独验证：微信开发者工具编译、真机购买/退款/扫码/拍照、云端同版部署、`lifecycleTick` 定时器、真实支付、真实退款、订阅消息真机发送和数据库迁移/备份。
