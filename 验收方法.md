# V1.6 验收方法

## 1. 自动检查

依次运行 README 中的领域、订单、批次、履约、C端和商家端测试，最后运行：

```bash
node scripts/check-integrity.js
```

预期全部PASS，且完整性检查覆盖JS语法、JSON解析、WXML标签、UTF-8、旧规则和前后端action对应关系。

## 2. 边界与竞争

- 北京时间21:59:59/22:00下单与支付边界。
- 取货日11:59:59/12:00逐站确认边界。
- 最后一件库存的并发预占。
- 支付与15分钟超时竞争。
- 用户退款与核销/固定放置竞争。
- 退款失败后保持`关闭退款中`并可由生命周期续跑。

## 3. 14页面跳转

按以下主路径逐页检查：品牌首页 → 点单 → 商品详情 → 选站/手机号 → MOCK支付 → 支付成功 → 我的订单 → 订单详情；再检查我的、分享落地点单、商家首页、商品、批次、站点和现场核销页。

## 4. 权限与隐私

- 普通用户不能初始化演示数据或自绑定管理员。
- 核销员只看到授权站点与对应手机号。
- 普通核销员不能跨站；超级管理员跨站必须确认、原因和日志。
- 核销员不能进入商品、批次、站点和退款管理。

## 5. 云端与真机

在微信开发者工具部署云函数和集合后，用至少两台真机演练并检查云数据库事务结果。当前`MOCK_PAY=true`仅验收内部演示；正式上线还需独立完成微信支付/退款回调、订阅消息和商户配置联调。
